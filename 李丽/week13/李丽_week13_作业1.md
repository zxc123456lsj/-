# Deepseek-V3相较于V2的改进
DeepSeek-V3 是一款拥有 671B 参数量（激活 37B）的混合专家（MoE）语言模型。相比于 V2，其核心改进主要体现在架构优化、训练目标及工程实现三个方面：

架构创新：无辅助损失的负载均衡策略 (Auxiliary-Loss-Free Load Balancing) V3 沿用了 V2 的多头潜在注意力（MLA）和 DeepSeekMoE 架构，但首创了无辅助损失的负载均衡策略。该策略消除了传统 MoE 模型为了平衡专家负载而引入的辅助损失对模型性能的负面影响。
训练目标：引入多 Token 预测 (Multi-Token Prediction) 在预训练阶段，V3 摒弃了传统的单 Token 预测，采用了多 Token 预测作为训练目标。这一改进显著增强了模型在各项基准测试中的整体表现，并提升了推理效率。
训练框架：支持 FP8 混合精度与 DualPipe 流水线并行 V3 首次在超大规模模型上成功验证了 FP8 混合精度训练的有效性，大幅降低了显存占用并加速了计算。同时，引入了 DualPipe 算法，通过计算与通信的完美重叠，掩盖了跨节点全对全（All-to-All）通信的开销。
极高的训练稳定性与低成本 在 14.8T 高质量 Token 的预训练过程中，模型未出现任何不可恢复的 Loss 峰值或回滚（Rollbacks）。整个全量训练仅耗费 2.788M H800 GPU 小时。
上下文能力扩展 通过两阶段的扩展策略，V3 的最大上下文支持长度从 V2 的水平进一步提升至 128K。
