# 多模态 RAG 聊天机器人 - 需求文档

## 1. 项目背景

在信息爆炸的时代，大量知识存在于图文混排的文档、报告和网页中。传统纯文本问答系统难以处理需要结合图像内容进行推理的复杂问题，例如：
- "这张图片里，左边那个设备的功能是什么？"
- "根据图表显示，产品A的销售额在哪个季度开始下降？"

本项目构建一个**多模态 RAG（Retrieval-Augmented Generation）聊天机器人**，能够：
- 理解图文混排的 PDF 文档
- 同时检索相关的文本和图像
- 基于检索结果进行多模态推理并生成答案

## 2. 功能需求

### 2.1 文档上传与管理
| 功能 | 描述 |
|------|------|
| 文件上传 | 支持上传 PDF、DOCX、TXT 文件 |
| 文件存储 | 本地存储原始文件，SQLite 记录元信息 |
| 文件列表 | 展示已上传文件，支持删除 |
| 状态追踪 | 记录文件处理状态（已上传/解析中/已完成） |

### 2.2 文档解析与索引（异步离线处理）
| 功能 | 描述 |
|------|------|
| PDF 解析 | 使用 MinerU 将 PDF 解析为 Markdown + 图片 |
| 文本分块 | 将 Markdown 按段落/长度切分为 chunks |
| 向量编码 | 使用 BGE 编码文本，CLIP 编码文本+图片 |
| 向量存储 | 存储到 Milvus 向量数据库 |

### 2.3 多模态问答
| 功能 | 描述 |
|------|------|
| 用户提问 | 支持纯文本提问 |
| 向量检索 | 基于 BGE 编码查询，检索相关文本 chunks |
| 答案生成 | 使用 Qwen-Plus 基于检索内容生成答案 |
| 图片展示 | 答案中引用的图片直接渲染显示 |

## 5. 数据模型

### 5.1 SQLite 文件表
```python
class File(Base):
    __tablename__ = 'files'
    
    id = Column(Integer, primary_key=True)
    filename = Column(String(255), nullable=False)
    filepath = Column(String(1000), nullable=False)
    filestate = Column(String(20), nullable=False)  # 已上传/解析中/已完成/失败
    created_at = Column(DateTime, default=datetime.now)
```

### 5.2 Milvus Collection 设计
```python
Collection: rag_data

Fields:
- id: INT64 (主键, auto_id=True)
- text_vector: FLOAT_VECTOR (512维, BGE编码)
- clip_text_vector: FLOAT_VECTOR (1024维, CLIP文本编码)
- clip_image_vector: FLOAT_VECTOR (1024维, CLIP图像编码)
- text: VARCHAR (文本内容)
- db_id: INT64 (关联的文件ID)
- file_name: VARCHAR (文件名)
- file_path: VARCHAR (文件路径)
```

## 6. 核心流程

### 6.1 文档上传流程
```
1. 用户上传 PDF 文件
2. 保存到 ./uploads/{uuid}.pdf
3. SQLite 插入文件记录（状态：已上传）
4. 发送 Kafka 消息到 rag-data topic
5. 返回上传成功响应
```

### 6.2 离线处理流程
```
1. Worker 消费 Kafka 消息
2. 调用 MinerU 解析 PDF：
   mineru -p {file_path} -o ./processed -b vlm-http-client
3. 读取生成的 Markdown 文件
4. 文本分块（按段落，最大256字符）
5. 对每个 chunk：
   - BGE 编码文本 → text_vector
   - CLIP 编码文本 → clip_text_vector
   - CLIP 编码关联图片 → clip_image_vector
6. 插入 Milvus
7. 更新 SQLite 状态为"已完成"
```

### 6.3 问答流程
```
1. 用户输入问题
2. BGE 编码问题 → 向量
3. Milvus 搜索相似文本（top-5）
4. 组装 prompt：
   - 系统指令：基于资料回答问题
   - 上下文：检索到的文本 chunks
   - 问题：用户提问
5. 调用 Qwen-Plus 生成答案
6. 渲染答案（处理 Markdown 图片语法）
```

## 8. 非功能需求

### 8.1 性能要求
- 文件上传：< 2s
- 文档解析：异步处理，不阻塞上传
- 问答响应：< 5s（包含检索和生成）

### 8.2 可靠性
- Kafka 消息持久化，Worker 故障可恢复
- 解析失败记录状态，支持重试

### 8.3 扩展性
- 支持多个 Worker 并发消费
- 向量数据库支持水平扩展
