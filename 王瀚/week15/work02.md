## MinerU 与 Pdfplumber 对比

### 📊 核心差异对比

| 对比维度 | **MinerU (Web/客户端)** | **Pdfplumber** |
| :--- | :--- | :--- |
| **技术架构** | **基于模型的解析一体库** (AI驱动) | **传统PDF解析库** (规则驱动) |
| **表格提取** | **整页/复杂表格**擅长处理跨页、合并单元格、无边框表格。 | **单表/规则表格**依赖线条检测，对复杂表格（如嵌套、无线条）处理较弱。 |
| **文本顺序** | **高度保留**利用AI理解阅读顺序，对多栏、图文混排还原度高。 | **基础保留**按PDF底层流顺序提取，多栏文档容易出现文本错乱。 |
| **特殊内容** | **支持OCR**可识别扫描件、图片中的文字；能处理化学分子式、复杂数学公式。 | **不支持OCR**仅能提取原生文本，无法处理图片和扫描件。 |
| **使用场景** | **RAG/知识库构建**适合需要高精度还原、处理复杂版式和扫描件的场景。 | **数据报表提取**适合提取规则清晰、以数据为主的简单表格。 |

### 💡 深度解析

#### 1. 技术路线的本质区别
*   **Pdfplumber** 是基于 `pdfminer` 的二次开发，属于**传统解析器**。它的工作原理是读取 PDF 中的绘图指令（线条、文本坐标），然后通过计算几何关系来提取内容。因此，它对表格的提取严重依赖于表格是否存在明确的“线条”。
*   **MinerU** 属于**基于大模型的智能解析器**。它更像是一个“人”，通过视觉理解（OCR + Layout Analysis）来识别页面上的标题、段落、图表和表格。这使得它能像人眼一样“看懂”文档结构，而不只是读取代码。

#### 2. 复杂场景表现差异
根据 MinerU 官方提供的测试案例（如证券研究报告、化学论文、法律书籍）：
*   **表格处理**：MinerU 能够处理“整页表格”和“少线表”（即只有横线没有竖线的表格），而 Pdfplumber 在面对无线条表格时往往束手无策，或者需要编写极其复杂的正则表达式和坐标定位代码才能勉强提取。
*   **公式与特殊符号**：MinerU 对 LaTeX 公式、化学结构式有专门的识别优化，而 Pdfplumber 通常只能提取出乱码或断裂的文本。

#### 3. 易用性与成本
*   **Pdfplumber**：开源免费，适合开发者进行轻量级的脚本编写，但面对复杂文档需要大量的“调参”工作（调整精度、捕捉网格等）。
*   **MinerU**：提供 Web 端和客户端（目前有免费额度，如单日5000份，单文件≤200页），开箱即用，无需编写代码即可获得高质量的 Markdown 或 JSON 结果，更适合非技术人员或追求效率的团队。
