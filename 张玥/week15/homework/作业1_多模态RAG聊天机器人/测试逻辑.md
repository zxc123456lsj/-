# 多模态 RAG 聊天机器人测试逻辑

## 1. 静态检查

目标：确保代码没有语法错误。

测试命令：

```powershell
D:\AI_study_env\miniconda3\envs\py312\python.exe -m py_compile rag_core.py app.py run_local_demo.py
```

预期结果：

- 命令无报错。

## 2. 本地最小流程测试

目标：验证不依赖 Kafka/Milvus 的本地链路能跑通。

测试命令：

```powershell
D:\AI_study_env\miniconda3\envs\py312\python.exe run_local_demo.py
```

预期流程：

```text
上传示例文档
  -> 生成 SQLite 记录
  -> 解析为 Markdown
  -> 构建索引
  -> 提问检索
  -> 输出图文回答
```

预期结果：

- 控制台打印文档 ID。
- 控制台打印检索到的片段。
- 回答中保留 Markdown 图片链接。

## 3. 上传接口测试

目标：验证 `/upload/document` 能接收文件。

测试方式：

- 启动 FastAPI 服务。
- 使用 Swagger UI 或 curl 上传 `sample_data/sample_report.md`。

预期结果：

- 返回 `document_id`。
- SQLite 中新增记录。
- 文件保存到 `storage/uploads`。

## 4. 解析接口测试

目标：验证 `/worker/parse/{document_id}` 能把上传文件处理成 Markdown。

测试方式：

- 使用上传接口得到的 `document_id`。
- 调用解析接口。

预期结果：

- 文档状态更新为 `parsed`。
- 生成 `storage/processed/{document_id}/content.md`。

## 5. 索引构建测试

目标：验证 `/index/build` 能扫描已解析文档。

预期结果：

- 返回 chunk 数量。
- 已解析文档状态更新为 `indexed`。

## 6. 聊天问答测试

目标：验证 `/chat` 能根据问题检索相关资料。

测试问题：

```text
系统架构图说明了哪些模块？
```

预期结果：

- 返回命中的 chunk。
- 返回回答文本。
- 如果资料中有图片，回答里保留 `![...](...)`。

## 7. 异常测试

### 文件不存在

输入不存在的文件路径调用上传函数。

预期：

- 抛出清晰的 `FileNotFoundError`。

### 文档未解析就问答

没有构建索引时调用 `chat_with_rag`。

预期：

- 返回“没有可检索资料”的提示。

### 无检索结果

提问与知识库完全无关的问题。

预期：

- 返回低相关度结果或提示需要补充资料。

### 图片 Markdown 渲染

输入包含：

```markdown
请参考下图：![架构图](images/demo.png)
```

预期：

- `render_markdown_with_images` 能拆分出文本片段和图片片段。
