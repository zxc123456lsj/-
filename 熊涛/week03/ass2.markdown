## Regex Rule
### 优点：
1. 速度快
2. 可解释性强
3. 对高频和强模式意图非常稳定
4. 零训练成本，不依赖数据
### 缺点：
1. 覆盖能力弱
2. 维护成本随规则数指数级上升
3. 无法理解语义

## TF-IDF
### 优点：
1. 训练成本低，不需要GPU，数据要求较小
2. 推理速度快
3. 效果稳定、易调参
4. 作为 baseline 非常合适
### 缺点：
1. 无法建模语义
2. 对口语、同义表达不鲁棒
3. 上限明显

## BERT
### 优点：
1. 强语义理解能力
2. 在意图识别任务上效果非常好
3. 对数据噪声更鲁棒
4. 工业界成熟方案
### 缺点：
1. 推理延迟相对高，需要蒸馏和量化
2. 资源消耗较大
3. 训练和部署复杂度高

## LLM
### 优点：
1. 极强的泛化能力
2. 几乎不依赖标注数据
3. 适合复杂业务场景
### 缺点：
1. 延迟高
2. 成本高
3. 可控性差
