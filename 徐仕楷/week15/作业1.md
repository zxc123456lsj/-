# 多模态 RAG Chatbot 框架说明

## 1. 项目概述

本项目实现了一个多模态检索增强生成（RAG）系统，能够处理图文混排的 PDF 文档，支持用户通过自然语言提问，并结合图像和文本内容生成准确答案。

### 核心能力
- **多模态理解**：同时理解用户提问和知识库中的图像、文本
- **跨模态检索**：从 PDF 文档中检索相关图像、文本段落
- **图文关联推理**：融合图像和文本信息进行逻辑推理
- **答案生成**：基于检索结果生成答案，并标注信息来源

## 2. 系统架构

```
┌─────────────────┐     ┌─────────────┐     ┌──────────────────────────┐
│   FastAPI        │────▶│   Kafka     │────▶│  offline_process_worker  │
│  /upload/document│     │ "rag-data"  │     │  (Consumer)              │
└─────────────────┘     └─────────────┘     └──────────────────────────┘
                                                        │
                                                        ▼
┌─────────────────┐     ┌─────────────┐     ┌──────────────────────────┐
│  /chat          │────▶│   Milvus    │◀────│  mineru PDF 解析           │
│  (Streamlit)    │     │ rag_data_new│     │  BGE/CLIP 向量化           │
└─────────────────┘     └─────────────┘     └──────────────────────────┘
```

### 组件说明

| 组件 | 职责 |
|------|------|
| **FastAPI (app.py)** | 提供 REST API：上传文件、问答、文件管理 |
| **Kafka** | 消息队列，解耦文档上传和解析处理 |
| **offline_process_worker** | Kafka 消费者，离线解析 PDF、生成向量 |
| **Milvus** | 向量数据库，存储 text/clip  embeddings |
| **mineru** | PDF 解析工具，输出 Markdown + 图片 |
| **BGE** | 文本向量化模型 (bge-small-zh-v1.5) |
| **CLIP** | 多模态向量化模型 (jina-clip-v2) |
| **LLM** | 答案生成 (DeepSeek-V4-Pro) |

## 4. API 接口

### 4.1 POST /upload/document
上传文档到指定知识库。

```bash
curl -X POST "http://localhost:8000/upload/document" \
  -F "file=@document.pdf" \
  -F "kb_id=1"
```

**Response:**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "file_id": 123,
    "filename": "document.pdf",
    "state": "pending"
  }
}
```

**流程:**
1. 保存文件到 `uploads/{uuid}.pdf`
2. 写入 `files` 表，状态为 `pending`
3. 发送 Kafka 消息到 `rag-data` topic

---

### 4.2 POST /chat
多模态问答接口。

```bash
curl -X POST "http://localhost:8000/chat" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "这张图片里左边设备的功能是什么？", "kb_id": 1, "top_k": 5}'
```

**Response:**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "answer": "根据文档分析，左边设备是...",
    "sources": [
      {"file": "document.pdf", "text": "设备功能说明..."}
    ]
  }
}
```

**流程:**
1. BGE 编码用户提问 → 向量
2. Milvus 检索 `text_vector`，召回 top_k 结果
3. 整理检索到的文本和图像路径
4. 调用 LLM 生成答案

---

### 4.3 GET /files
查询知识库中的文件列表。

```bash
curl "http://localhost:8000/files?kb_id=1"
```

---

### 4.4 DELETE /files/{file_id}
删除文件及其在 Milvus 中的向量。

```bash
curl -X DELETE "http://localhost:8000/files/123"
```

## 5. 数据模型

### 5.1 SQLite - files 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| filename | VARCHAR(255) | 原始文件名 |
| filepath | VARCHAR(1000) | 本地存储路径 |
| filestate | VARCHAR(20) | pending/processing/done |
| kb_id | INTEGER | 知识库ID |

### 5.2 Milvus - rag_data_new collection

| 字段 | 类型 | 维度 | 说明 |
|------|------|------|------|
| id | INT64 | - | 主键 (auto) |
| text_vector | FLOAT_VECTOR | 512 | BGE 文本向量 |
| clip_text_vector | FLOAT_VECTOR | 1024 | CLIP 文本向量 |
| clip_image_vector | FLOAT_VECTOR | 1024 | CLIP 图像向量 |
| text | TEXT | - | 文本内容 (chunk) |
| db_id | INT64 | - | 对应 files.id |
| file_name | VARCHAR(255) | - | 文件名 |
| file_path | VARCHAR(1000) | - | 原始文件路径 |

## 6. Worker 处理流程

`offline_precess_worker.py` 作为 Kafka 消费者：

1. **消费消息** - 从 `rag-data` topic 接收文件信息
2. **PDF 解析** - 调用 mineru：
   ```bash
   mineru -p {file_path} -o ./processed -b vlm-http-client -u http://127.0.0.1:30000
   ```
   输出 Markdown + images 目录
3. **文本分块** - 按 256 字符分割成 chunks
4. **向量化** - 每块文本生成：
   - BGE 向量 (text_vector)
   - CLIP 向量 (clip_text_vector)
   - 关联图像的 CLIP 向量 (clip_image_vector)
5. **向量存储** - 批量插入 Milvus
